Sandbox Orchestration & Secure Live Channel

Visual explainer for the MCP-based bidirectional channel between AIDevOps and the in-container Claude agent. Companion to SANDBOX-ORCHESTRATION.md.

The question this answers

Can AIDevOps talk live to Claude inside a sandbox container — progress, mid-run context, approvals, intermediate artifacts — without the channel becoming an exploit surface?

Short answer: Yes. MCP Streamable HTTP + OAuth 2.1 audience-bound per-run bearer tokens + sandbox-dials-out-only + defense-in-depth. Same pattern GitHub Actions, GitLab Runner, Temporal, and Argo Workflows use.

Topology — outbound-only, no inbound port

Allowed outbound Blocked inbound Container boundary Orchestrator boundary

The container never listens. A compromised agent has no inbound route to be reached from. Egress is allowlisted to exactly two endpoints: AIDevOps MCP server + Anthropic API.

Per-run token lifecycle

Why this is safe: a leaked token cannot be replayed elsewhere (audience-bound to this MCP server URL via RFC 8707), cannot be used after the run (TTL expiry + revocation), cannot be expanded in scope (server enforces tool allowlist per token), and is never stored on disk in plaintext.

Tool surface — the 8 things AIDevOps exposes

report_progress

Heartbeat + structured progress signal
phase, percent: 0-100, message

submit_intermediate_artifact

Mid-run output (log, diff, screenshot)
kind: enum, content: bytes ≤10MB, metadata

request_human_approval

Gate a destructive op; blocks until human responds
action, rationale, blast_radius, timeout ≤3600s

get_more_context

Pull queued context (poll-based push)
query, kind: enum

query_workflow_state

Current plan / step / dependencies
(no args)

log_event

Structured telemetry
level: enum, event, fields: object

register_finding

Record an issue/risk discovered mid-run
severity: enum, title, evidence

notify_completion

Terminal signal with summary
status: enum, summary

Every tool accepts an optional client_event_id for idempotency under retry. Rate limits: ~100 calls/minute, ~1000 calls/run, per-token-bucket.

Phases 0–6 are byte-for-byte identical. The AIDevOps path is a wrapper around the CLI, not a parallel implementation. Three new CLI flags cover it:

--mcp-endpoint <url>           # AIDevOps MCP server URL
--mcp-token-file <path>        # path to per-run bearer token file
--egress-allowlist <hosts>     # docker network egress allowlist

The killer property: with MCP, Claude already knows how to call the tools. We don't have to put the API contract in the system prompt — the schema is the contract. Every alternative forces a prompt-encoded contract, which re-opens the prompt-injection attack surface we're trying to shrink.
